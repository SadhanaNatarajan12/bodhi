ClickJacking


Clickjacking, also known as a "UI redress attack", is when an attacker uses multiple transparent or opaque layers to trick a user into clicking on a button or link on another page when they were intending to click on the the top level page. Thus, the attacker is "hijacking" clicks meant for their page and routing them to another page, most likely owned by another application, domain, or both. For more information, please refer this.

Bodhi focuses on two types of clickjacking attacks: Single click & Double click.

Scenario


The application has 2 roles i.e Admin user & normal user. Admin user can update existing normal user to Admin. This update user page is vulnerable to clickjacking.

The objective is to expoit the Clickjacking vulnerability to update existing normal user to Admin user, log in using updated admin user credentials & read the flag.

Exploitation


Create a clickjacking PoC and upload to the attacker server (in this case, another server running on post 8000) using "upload" page. Admin user will visit PoC page while authenticated to the application and will click the buttons on the page. For creating PoC, you can refer this blogpost.

Note:

Admin user clicks buttons with specific IDs. For single click, button ID should be "button1". 
For double click, button IDs should be "button1" & "button2".
