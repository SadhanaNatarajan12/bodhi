Cross-Site Request Forgery (CSRF)


Cross-Site Request Forgery (CSRF) is an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated. With a little help of social engineering (such as sending a link via email or chat), an attacker may trick the users of a web application into executing actions of the attacker's choosing. For more information, please refer this.

The current release of this playground focusses on learning various types of Cross-Site Request Forgery (CSRF) attacks.

Scenario


The application has 2 roles i.e Admin user & normal user. Admin user can create other admin users and also can read the flag. The form to create new admin user is vulnerable to CSRF.

The objective is to expoit the CSRF vulnerability to create new admin user, log in using newly created admin credentials & read the flag.

The application provides 2 ways to exploit CSRF vulnerability:


Using Cross-Site Scripting (XSS):

The application provides functionality to send messages to Admin user. The page where these messages are rendered, is vulnerable to Stored Cross-Site Scripting (XSS) vulnerability. Admin user will always reading the messages sent to him while logged in into the application.


Using Different Site:

Normal user can upload the POC page to a different site (in this case, another server running on post 8000). Admin user will visit this page while logged in into application.
